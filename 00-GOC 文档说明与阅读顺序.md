# GOC 文档说明与阅读顺序

本文档说明当前 GOC 相关三个核心文档的标题、定位及相互关系，便于产品、开发与协作方统一理解。

---

## 一、三个文档的标题与定位

| 文档 | 标题 | 定位 |
|------|------|------|
| **文档 1** | **01-GOC 指令规范** | **语言标准**：定义 GOC 的语法、`pen` API、绘图指令、数据类型、错误与边界，供解析器、执行引擎、判题与文档统一遵循。 |
| **文档 2** | **02-GOC 在线编程网站** | **Web 产品设计**：页面结构、功能模块、交互流程、可扩展方向，描述用户看到的界面与操作路径。 |
| **文档 3** | **03-GOC 判题教学系统** | **完整教学/判题平台 PRD**：用户角色、系统架构、判题方式、题库、反馈与用户体系，描述整条「做题→判题→反馈」链路。 |

---

## 二、三者关系（从底层到产品）

```
┌─────────────────────────────────────────────────────────────┐
│  03-GOC 判题教学系统                                           │
│  完整平台：编辑 + 执行 + 判题 + 题库 + 用户 + 反馈               │
└───────────────────────────┬─────────────────────────────────┘
                            │ 依赖
┌───────────────────────────▼─────────────────────────────────┐
│  02-GOC 在线编程网站                                           │
│  前端产品：页面布局、绘图区、代码区、命令参考、运行/清屏等          │
└───────────────────────────┬─────────────────────────────────┘
                            │ 实现并遵循
┌───────────────────────────▼─────────────────────────────────┐
│  01-GOC 指令规范                                               │
│  语言定义：pen 链式调用、绘图指令、控制流、数据类型、错误规范       │
└─────────────────────────────────────────────────────────────┘
```

- **01-GOC 指令规范**：语言层。规定 GOC 有哪些指令、怎么写、什么算错；解析器、执行引擎、判题里的「执行轨迹」都按此规范实现。
- **02-GOC 在线编程网站**：产品/交互层。用户看到的页面、命令参考区里的 `fd/rt/pen` 等对应 01 的指令；编辑、运行、清屏、倍速等围绕「写 GOC、看图形」的体验设计。
- **03-GOC 判题教学系统**：系统层。在「网页编辑 + 运行」之上，增加编译/执行引擎、轨迹生成、判题（轨迹/图形/结构）、题库、用户与反馈，形成完整教学与自动判题平台；其「执行轨迹」格式（如 `{ "cmd": "fd", "args": [100] }`）即 01 中指令的规范化输出。

---

## 三、整体意图（一句话）

你在定义一套 **以 GOC 为核心的图形化编程教学体系**：先定语言（01），再定网页产品（02），最后定完整系统（03），三者是「规范 → 界面 → 平台」的递进关系。

---

## 四、建议阅读顺序

1. **先读 01-GOC 指令规范**：理解 GOC 能写什么、怎么执行。
2. **再读 02-GOC 在线编程网站**：理解用户界面与操作流程。
3. **最后读 03-GOC 判题教学系统**：理解整条教学与判题链路及系统模块。
