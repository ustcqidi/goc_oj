# 1. 产品概述

**产品名称**：GOC Online Judge 教学系统（图形化编程 + 判题平台）

**产品定位**：面向初学者和教育场景的在线编程平台，结合 **图形化 GOC 语言** 与 **自动判题系统**，提供即时可视化、交互式编程学习体验。

**核心价值**：

* 所见即所得：代码执行结果在绘图区实时呈现
* 学习驱动：题库 + 自动判分 + 可视化反馈
* AI 辅助：智能生成代码、辅助判题、可视化 Debug

# 3. 系统架构模块

## 3.1 在线编辑器模块

**功能点**：

* 代码编辑器（语法高亮、缩进）
* 自动补全 `.fd() / .rt() / pen.xxx()`
* 运行按钮：执行绘图
* 重置按钮：清空绘图区与输入输出
* 可选：多标签页（多个题目并行编辑）

---

## 3.2 编译与执行引擎

**功能点**：

* GOC 解析器（AST 生成）
* 绘图指令执行
* 生成执行轨迹：

```json
[
  { "cmd": "fd", "args": [100] },
  { "cmd": "rt", "args": [90] }
]
```

* 支持 step-by-step 执行
* 支持输入输出处理（cinWin / coutWin）

---

## 3.3 渲染引擎（Canvas）

**功能点**：

* 根据执行轨迹绘制图形
* 动画展示（速度可控）
* 支持抬笔/落笔、颜色、粗细、图形类型
* Step 模式：逐步显示执行结果

---

## 3.4 判题系统

### 判题方式（必须三种）

1. **轨迹判定**（主方案）

   * 对比执行轨迹 JSON
   * 支持容差（浮点 / 坐标）
2. **图形判定**

   * 可选：最终绘图区截图对比
   * 辅助轨迹判定
3. **结构判定**

   * 检查代码是否使用指定语法（如 for 循环）

### 判题输入

```json
{
  "expected": [...],
  "actual": [...]
}
```

### 判题策略

* 主策略：轨迹匹配
* 辅助策略：图像相似度
* 容差机制：±2 px 或角度误差 ±1°

---

## 3.5 题库系统

### 题目结构

```json
{
  "id": 1,
  "title": "画正方形",
  "description": "使用循环画一个边长100的正方形",
  "difficulty": "easy",
  "starterCode": "pen.size(2).c(1);",
  "solution": "for(int i=0;i<4;i++){ pen.fd(100).rt(90); }",
  "testcase": {
    "expectedTrajectory": [...]
  }
}
```

### 题目类型

* 入门：直线 / 移动
* 进阶：循环 / 图形组合
* 高级：复杂组合图形
* AI 创作题：开放题，AI 自动生成参考解

---

## 3.6 判分与反馈系统

* Pass / Fail
* 错误类型：

  * 图形错误：最终图形不符合
  * 行为错误：执行步骤错误
  * 结构错误：缺少指定语法
* 可视化反馈：

  * 叠加预期轨迹与实际轨迹
  * 高亮错误步骤
* 建议修复：

  * 自动提示修改步骤
  * AI 推荐修复代码

---

## 3.7 用户系统

* 登录 / 注册
* 做题记录与进度 tracking
* 统计信息（正确率、使用时间）
* 可选：排行榜 / 学习路径

# 4. 核心流程

## 4.1 学生做题流程

```text
打开题目 → 编写代码 → 运行 → 绘图区显示 → 查看输出 → 提交 → 判题 → 获取反馈 → 可修正
```

## 4.2 判题流程

```text
代码 → AST → 执行轨迹生成 → 对比 expected → Pass/Fail → 输出反馈
```

---

# 5. 关键设计点

1. **为什么用轨迹判题**

   * 更精准、可量化
   * 可支持 step-by-step 反馈
   * 支持容错和浮点误差

2. **误差处理**

   * 坐标容差 ±2px
   * 角度误差 ±1°
   * 保证教学容忍度，不影响学习

3. **防作弊**

   * 禁止硬编码
   * 判题时比对轨迹，而非单纯输出
   * 支持 AI 自动检查重复解

4. **AI 支持出题与解题**

   * 自动生成题目（轨迹 + 描述 + 测试用例）
   * AI 自动生成参考解
   * AI 辅助学生调试
