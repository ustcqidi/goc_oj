# GOC 系统实现缺口清单（AI 开发用）

在已有 01-GOC 指令规范、02-GOC 在线编程网站、03-GOC 判题教学系统 三份 PRD 的基础上，若由 AI 或开发团队实现整条系统，还缺少以下文档或规格。按模块列出，便于补齐后直接用于实现。

---

## 一、技术架构与选型

| 缺口 | 说明 | 建议产出物 |
|------|------|------------|
| **技术架构说明** | 前端/后端技术栈、部署方式、模块边界 | `05-技术架构与选型.md`：前端（如 React/Vue + Canvas）、后端语言与框架、是否 BFF、执行引擎单独服务等 |
| **执行引擎部署方式** | GOC 解析与执行是浏览器内（WebAssembly/JS）还是服务端 | 在架构文档中明确：影响判题时是「前端跑完上传轨迹」还是「服务端跑代码」 |

---

## 二、执行引擎与轨迹

| 缺口 | 说明 | 建议产出物 |
|------|------|------------|
| **执行轨迹 JSON 规范** | 03 中提到的轨迹格式未完整定义：字段名、单位、顺序、可选字段 | `06-执行轨迹规范.md`：例如 `{ "cmd", "args", "timestamp?" }`，以及 fd/rt/pen 等每条指令的标准化输出格式 |
| **GOC 解析器/编译器规格** | 01 只有语言规范，没有「如何从源码到 AST、再到轨迹」的算法说明 | `07-GOC 解析与执行引擎规格.md`：词法/语法规则、AST 结构、执行顺序、与 01 指令的一一对应 |
| **容差与判题参数** | 03 提到 ±2px、±1°，未集中成可配置的数值与规则 | 在判题规格或配置中：坐标/角度容差、超时时间、步数上限等 |

---

## 三、判题系统

| 缺口 | 说明 | 建议产出物 |
|------|------|------------|
| **判题接口规格** | 输入（代码/语言/题目 ID）、输出（Pass/Fail、错误类型、反馈内容）、侧端（轨迹/截图） | `08-判题服务 API 与数据格式.md`：请求/响应 JSON、错误码、与 06 轨迹格式的对应 |
| **轨迹比对算法** | 如何判定 expected 与 actual 轨迹「匹配」（顺序、容差、忽略无关指令等） | 在 08 或单独 `09-轨迹比对与判题逻辑.md` 中描述或伪代码 |
| **图形/结构判定规则** | 图形判定（截图对比）与结构判定（必须含 for 等）的具体规则与优先级 | 在 08 或 09 中补充，便于实现与测试 |

---

## 四、数据与存储

| 缺口 | 说明 | 建议产出物 |
|------|------|------------|
| **题库数据模型** | 03 有题目 JSON 示例，缺完整 schema：字段必填/可选、类型、与判题用例的关联 | `10-题库与题目数据模型.md`：题目、测试用例、expectedTrajectory 等表结构或 JSON Schema |
| **用户与进度数据模型** | 用户、提交记录、做题进度、统计等 | `11-用户与提交数据模型.md` 或合并到 10 |
| **API 列表与鉴权** | 除判题外，题目列表、提交历史、用户信息等接口与鉴权方式 | 在架构文档或单独 `12-后端 API 列表与鉴权.md` 中列出 |

---

## 五、前端与交互

| 缺口 | 说明 | 建议产出物 |
|------|------|------------|
| **页面与路由清单** | 02 描述了单页布局，未明确多页/路由（题目列表、题目详情、提交结果、用户中心等） | 在 02 补充或 `13-前端页面与路由.md` |
| **编辑器与绘图区接口** | 代码编辑器与绘图区之间的数据流：代码 → 运行 → 轨迹 → 绘图区渲染；是否支持 step 模式 | `14-前端模块接口说明.md`：与执行引擎/判题的数据交换格式与时机 |
| **错误与状态码在前端的展示** | 编译错误、运行错误、判题结果等如何映射到 UI 文案与样式 | 在 02 或 14 中简要约定，便于前后端对齐 |

---

## 六、开发与交付

| 缺口 | 说明 | 建议产出物 |
|------|------|------------|
| **实现优先级与里程碑** | 先做「仅运行+展示」还是「运行+判题」、题库与用户系统的先后顺序 | `15-实现优先级与里程碑.md` 或写在 00 说明中 |
| **测试用例与验收标准** | 针对 01 指令、判题逻辑、关键流程的测试用例或验收清单 | 可分散在 06/08/09，或单独 `16-测试与验收清单.md` |
| **环境与配置** | 开发/测试/生产环境、环境变量、依赖服务（如判题队列、存储） | `17-环境与配置说明.md` 或 README |
