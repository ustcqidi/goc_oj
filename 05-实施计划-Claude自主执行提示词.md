# GOC 阶段一实施 — Claude 自主执行提示词

本文档提供可直接复制给 Claude 的提示词，用于**在无人值守情况下**完成《05-实施计划》阶段一全部步骤：自主分析需求、补全文档、拆分任务、按序执行、验收成果。你只需在会话开始时粘贴对应提示词即可。

**建议**：在 Cursor/Claude 中打开本项目并确保 `goc/` 目录在上下文中，以便 Claude 能直接读取 01、03、05、06 和 `problems/samples/` 下的文件。

**若你想「出题 + OJ 一起做完」**：见同目录下的 **《05-实施计划-出题与OJ一次性完成.md》**，其中有 OJ MVP 范围说明和「出题 + OJ MVP 一次性执行」的完整提示词（纯前端、无登录、题库静态 JSON、浏览器内执行与判题）。

## 最小化提示词（一句话启动）

若你只想发一句就给 Claude 跑完全程，可用下面这句（执行范围与「提示词 A」一致）：

```
请按《05-实施计划（先出题后 OJ）.md》阶段一，从 Step 2 执行到 Step 6（Step 1 已有 06 与 samples），自主完成 07/08/09 文档、在 problems/ 下生成 10～20 道题、并生成 index；不要问我选择，自己做合理决策；最后按交付清单做验收并输出报告。
```

## 使用方式

1. **一次性跑完全程**：复制「提示词 A：总控任务」整段发给 Claude，让其从 Step 2 执行到 Step 6 并做最终验收。
2. **分步执行**：若希望每步确认后再继续，可依次使用「提示词 B：单步执行」中的 Step 2～6。
3. **仅验收**：若你认为已做完但想再检查，使用「提示词 C：验收与报告」。

## 提示词 A：总控任务（建议首次发送）

```
你是 GOC 阶段一实施的执行者，需要在**无需我进一步输入**的前提下，独立完成《05-实施计划（先出题后 OJ）.md》中阶段一的全部工作。

【当前状态】
- Step 1 已完成：`06-GOC 题目案例与模式提炼.md` 与 `problems/samples/` 下已有样例题目（001～003），出题模式已部分提炼。
- 你需要完成的是：Step 2 → Step 3 → Step 4 → Step 5 → Step 6，并做最终验收。

【你必须先做的事】
1. 阅读并理解以下文件（按顺序）：
   - `05-实施计划（先出题后 OJ）.md`（阶段一步骤与交付清单）
   - `06-GOC 题目案例与模式提炼.md`（题目模式与样例分析）
   - `01-GOC 指令规范.md`（GOC 语法与指令）
   - `03-GOC 判题教学系统.md`（题目结构、判题与题库约定）
   - `problems/samples/` 下各题的 `description.md`（可选：看 output.png 理解图形）
2. 根据 05 中的「阶段一详细步骤」和「阶段一交付清单」，在本地列出你要产出的文件与目录，形成自己的任务清单。

【执行顺序与要求】
- **Step 2**：编写 `07-GOC 题目数据格式（Schema）.md`。依据 06 的样例形态与 03 的题目结构，定义单道题目的标准格式（必填/可选字段、JSON 示例、与 06/03 一致）。若 03 与 06 有差异，以 06 和 samples 的实际形态为准。
- **Step 3**：编写 `08-GOC 题目分类与难度体系.md`。难度与标签需与 06 中已提炼的题型、难度标志一致，并补充完整分类表与数学对应关系。
- **Step 4**：编写 `09-GOC 出题专家规范（出题指南）.md`。以 06 的案例题干风格和 05 中的出题原则为范本，写出题干写法、难度控制、与数学结合、判题要点等，并给出出题 checklist。
- **Step 5**：在 `problems/` 目录下（注意：不是 `problems/samples/`）按 07 的 Schema 生成**至少 10 道、至多 20 道**新题目。要求：
  - 每题一个文件，命名 `编号-题目名.json`（如 `001-画正方形.json`），编号从 001 起连续；题目名用简短中文。
  - 内容包含：id、title、description、difficulty、tags、starterCode、solution；expectedTrajectory 可留空或省略。
  - 难度与题型分布：以入门为主（约 6～8 道），进阶次之（约 3～5 道），可有 1～2 道高级；需覆盖 06/08 中的不同标签（如循环、几何、正多边形、等差数列等），并与小学初中数学有对应。
  - 题目表述与图形设计要**多样、不雷同**：不要照抄 06 的模式句，同一知识点用不同情境或图形呈现。
- **Step 6**：根据 `problems/` 下已生成的题目文件，生成 `problems/index.json`（或 `problems/README.md`）。索引需包含每题 id、title、difficulty、tags、文件路径（或文件名），便于按难度/标签筛选与后续 OJ 导入。

【决策原则】
- 遇到 05/06/03 未明确约定的细节（如题目存 JSON 还是 MD、id 用数字还是字符串），由你做出**合理选择**并在 07 中写清楚，后续步骤一律遵循该选择。
- 不要向我提问「要 JSON 还是 MD」等；直接选一种（建议题目用 JSON、07 中给出 JSON 示例），并保持全库一致。
- 若某一步依赖上一步的某个未写明的细节，用常识补全并保证 06/07/08/09 之间描述一致。

【最终验收】
- 全部完成后，按《05-实施计划》中「三、阶段一交付清单」的 6 条逐项自检：对应文件或目录是否存在、内容是否满足该条说明。
- 输出一份**验收报告**：每条交付物一行，标出「已交付」或「未交付」及路径；若有未交付或与计划不符的，简要说明原因或补救措施。
```

## 提示词 B：单步执行（按需使用）

```

### Step 3 专用

```
请仅完成《05-实施计划》中的 **Step 3**：建立题目分类与难度体系。

请先阅读：05（Step 3）、06（已提炼的题型与难度标志）。然后编写 `08-GOC 题目分类与难度体系.md`，包含难度定义（入门/进阶/高级）、知识点/标签表、与小学初中数学的对应，并与 06 中的案例难度和题型一致。
```

### Step 4 专用

```
请仅完成《05-实施计划》中的 **Step 4**：编写出题专家规范（出题指南）。

请先阅读：05（Step 4）、06（题干风格与出题原则）。然后编写 `09-GOC 出题专家规范（出题指南）.md`，包含题目形态、难度控制、与数学结合、01 指令集约束、solution/starterCode 与判题要点，以及出题 checklist。以 06 案例为范本，不向我提问，直接产出完整文档。

### Step 5 专用

```
请仅完成《05-实施计划》中的 **Step 5**：批量生成题目。

前提：07（Schema）、08（分类与难度）、09（出题指南）已存在；若不存在请先创建它们。

请在 `problems/` 目录下（不要放到 problems/samples/）按 07 的 Schema 生成 10～20 道新题，每题一个 JSON 文件，命名 `编号-题目名.json`，编号从 001 起。要求：入门为主、进阶次之、少量高级；覆盖 08 中多种标签；题干与图形设计多样、不照抄 06 的表述。每道题必须包含 id、title、description、difficulty、tags、starterCode、solution。
```

### Step 6 专用

```
请仅完成《05-实施计划》中的 **Step 6**：题目索引。

请根据 `problems/` 目录下已有的题目文件（仅统计根目录下的题目 JSON，不包含 samples 子目录），生成 `problems/index.json`（或 `problems/README.md`），包含每题 id、title、difficulty、tags、文件名或路径，便于按难度/标签筛选与后续 OJ 导入。
```

## 提示词 D：断点续跑（执行中断后继续）

```

1. 列出阶段一交付清单中的 6 项交付物，并检查当前 goc 目录下哪些已存在、哪些缺失。

3. 完成后输出简短结论：已补全的交付物列表 + 最终验收报告（同提示词 C 的格式）。
```
