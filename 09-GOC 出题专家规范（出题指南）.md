# GOC 出题专家规范（出题指南）

本文档规定 GOC 题目的出题标准，供人工或 AI 出题时对照，确保题目质量一致、难度合理、与数学结合紧密。

---

## 一、题干（description）写法

### 1.1 结构规范

题干必须包含以下要素：

1. **任务说明**（一句话）：用「画……」或「用……画出……」开头，点明目标图形
2. **绘图要求**（清单式）：
   - 颜色（指定 `pen.c(n)` 的值及颜色名）
   - 线条粗细（不改变时写「默认（1）」）
   - 图形的数量规律（边数、循环次数、角度规律）
   - 长度/半径规律（固定值或等差递增规律）
   - 是否有特殊约束（如「必须使用 for 循环」）

### 1.2 风格参考（以样例题为范本）

**样例 002 风格**（已有题目描述）：
```
画下面正方形螺旋线，长度 0、10、20、30…190

绘图要求：
- 颜色：蓝色（pen.c(2)）
- 线条粗细：默认（1）
- 转角：每次右转 90°
- 边长规律：第 i 条边长度为 i * 10（i 从 0 到 19，共 20 条）
- 必须使用 for 循环完成
```

### 1.3 禁止事项

- ❌ 不要照抄样例题的句式（换情境、换图形、换数学角度）
- ❌ 不要使用 01 规范以外的指令
- ❌ 不要在题干中直接给出解题代码或关键循环结构
- ❌ 不要让同一批题中两道题的图形类型 + 数学考点完全相同

## 二、难度控制

### 2.1 入门题要求

- 循环参数直接给出（如「每次右转 90°」「共 4 次」）
- 若用循环变量，规律要直观（如「第 i 条边长度为 i×10」）
- 角度值直接告知（不需要学生推导）
- solution 代码行数 ≤ 5 行（不含空行）
- **目标**：小学生看懂题目能写出来

### 2.2 进阶题要求

- 需要学生自行推导转角（如「6 条边均匀排一圈，转角是多少？」）
- 循环变量同时影响 2 个参数（边长 + 颜色，或外层 + 内层）
- 可能需要 `up/down/moveTo` 定位
- solution 代码行数 6～15 行
- **目标**：理解正多边形外角公式或等差数列

### 2.3 高级题要求

- 使用 `if/else` 判断（如奇偶行颜色不同）
- 双重循环（外圈控制数量 + 内圈控制形状）
- 或使用坐标绘制
- solution 代码行数 10～25 行
- **目标**：综合运用多个概念，有一定思维挑战性

## 三、与小学初中数学的结合

### 3.1 常用结合点

| 数学概念 | 出题方式 | 示例描述片段 |
|----------|----------|-------------|
| 正多边形外角 = 360°/n | 不直接给出转角，让学生思考 | 「画正六边形，每条边等长，共 6 条」→ 学生算出 rt(60) |
| 等差数列 | 描述规律而非公式 | 「长度依次为 10、20、30…100（每次增加 10）」 |
| 圆的周长 / 半径 | 同心圆、不同大小的圆 | 「画 5 个同心圆，半径分别为 20、40、60、80、100」 |
| 对称 | 描述对称结构 | 「从中心出发，均匀画 8 条射线，每条间隔 45°」 |
| 整除与余数 | if 分支颜色 | 「奇数编号用红色，偶数编号用蓝色」→ `i%2==0` |

### 3.2 避免的问题

- 不要出现超出 01 指令集的功能（如函数定义、数组）
- 不要要求精确输入/输出数字（GOC 是图形语言，不支持 cin/cout）
- 不要把数学公式直接写在题干里（让学生有一定思考空间）

## 四、solution 与 starterCode 规范

### 4.1 solution 要求

- 必须完整可运行，符合 01 语法（每句以 `;` 结尾，所有方法挂在 `pen` 上）
- 必须产出与题干描述完全一致的图形（颜色、方向、圈数等与描述吻合）
- 尽可能简洁，避免不必要的辅助变量
- 链式调用优先（`pen.fd(100).rt(90)` 而非分两句）

```cpp
// 好的 solution 写法
pen.c(2).size(2);
for(int i=0; i<6; i++){
    pen.fd(80).rt(60);
}
```

```cpp
// 避免这种写法（不必要的变量）
int side = 80;
int angle = 60;
pen.c(2);
pen.size(2);
for(int i=0; i<6; i++){
    pen.fd(side);
    pen.rt(angle);
}
```

### 4.2 starterCode 写法

- 入门题：可留空（`""`），或仅提供颜色设置 `pen.c(n);`
- 进阶题：可给出循环框架，留空参数
- 高级题：可给出变量声明框架

```cpp
// 进阶题起始代码示例
pen.c(2);
for(int i=0; i<___; i++){
    pen.fd(___).rt(___);
}
```

## 五、判题要点（为 OJ 预留）

每道题的 `expectedTrajectory` 由执行器跑 solution 生成，判题逻辑为：

- **基础判题**：用户代码跑出的轨迹与 expectedTrajectory 逐条比对，数值在容差内（坐标 ±2，角度 ±1°）
- **结构要求**（constraints 字段中说明）：
  - 「必须用 for 循环」：检查 solution 中是否含 for 关键字（可选，视 OJ 实现）
  - 目前 OJ MVP 只做轨迹比对，不做结构检查

## 六、出题 Checklist

每次出完一道题后，按以下清单逐项检查：

### 题干

- [ ] 任务说明清晰（「画什么图形」）
- [ ] 颜色已指定（`pen.c(n)` 及颜色名）
- [ ] 循环次数 / 角度规律已说明
- [ ] 边长 / 半径规律已说明
- [ ] 是否有额外约束（必须用 for 等）
- [ ] 没有直接给出解题代码

### 分类

- [ ] difficulty 已填（easy/medium/hard）
- [ ] tags 已从 08 标签表选取，至少 2 个
- [ ] 本批题中没有与其他题目完全重复的「图形类型 + 主要标签」组合

### solution

- [ ] 代码语法正确（所有语句以 `;` 结尾）
- [ ] 只使用 01 中已定义的指令
- [ ] 链式调用（尽量）
- [ ] 执行结果与题干描述完全一致
- [ ] 代码简洁（无冗余变量）

### JSON 格式

- [ ] id 与文件名编号一致
- [ ] 所有必填字段已填写（id, title, description, difficulty, tags, starterCode, solution）
- [ ] JSON 格式合法（可用 JSON.parse 验证）
- [ ] expectedTrajectory 已填（或暂留 `[]`）
- [ ] problems/index.json 已同步更新

## 七、题型多样性建议

批量出题时，参考以下题型清单，确保一批题不重复：

| 题型 | 代表图形 | 数学主题 |
|------|----------|----------|
| 螺旋多边形 | 螺旋三/四/六边形 | 等差数列 + 正多边形角度 |
| 正多边形 | 正三/四/五/六/八/十边形 | 正多边形外角 |
| 五角星 / 星形 | 五角星、六角星 | 星形角度（144°、120°）|
| 放射线 | 均匀放射 N 条射线 | 对称、等分 360° |
| 同心图形 | 同心圆、同心正方形 | 等差递增半径/边长 |
| 风车 / 花瓣 | N 个三角形绕中心旋转 | 对称、等分 360° |
| 渐变颜色 | 颜色随循环变化 | 模运算、等差数列 |
| 网格 | 横竖等间距线 | 坐标系、等差间隔 |
| 条件图案 | 奇偶颜色交替 | if/else、整除 |
| 楼梯 / 台阶 | 递增直角折线 | 等差、对称 |
