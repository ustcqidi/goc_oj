# GOC 执行轨迹规范

本文档定义 GOC 代码执行器输出的轨迹（Trajectory）格式，用于 OJ 判题时比对用户代码与参考答案的执行结果。

---

## 一、轨迹格式

轨迹是一个 **JSON 数组**，每个元素是一条执行指令记录：

```json
{ "cmd": "<指令名>", "args": [<参数列表>] }
```

- `cmd`：字符串，对应 01 规范中的 pen 方法名（小写）
- `args`：数组，按方法参数顺序排列，值为执行时的实际数值

### 示例：画正方形的轨迹

```cpp
pen.c(1).size(2);
for(int i=0; i<4; i++){
    pen.fd(100).rt(90);
}
```

生成的轨迹：

```json
[
  { "cmd": "c",    "args": [1]   },
  { "cmd": "size", "args": [2]   },
  { "cmd": "fd",   "args": [100] },
  { "cmd": "rt",   "args": [90]  },
  { "cmd": "fd",   "args": [100] },
  { "cmd": "rt",   "args": [90]  },
  { "cmd": "fd",   "args": [100] },
  { "cmd": "rt",   "args": [90]  },
  { "cmd": "fd",   "args": [100] },
  { "cmd": "rt",   "args": [90]  }
]
```

## 二、各指令对应的轨迹记录

| 01 指令 | cmd | args 说明 | 示例 |
|---------|-----|-----------|------|
| `pen.fd(n)` | `"fd"` | `[n]` | `{"cmd":"fd","args":[100]}` |
| `pen.bk(n)` | `"bk"` | `[n]` | `{"cmd":"bk","args":[50]}` |
| `pen.rt(angle)` | `"rt"` | `[angle]` | `{"cmd":"rt","args":[90]}` |
| `pen.lt(angle)` | `"lt"` | `[angle]` | `{"cmd":"lt","args":[60]}` |
| `pen.c(color)` | `"c"` | `[color]` | `{"cmd":"c","args":[1]}` |
| `pen.size(n)` | `"size"` | `[n]` | `{"cmd":"size","args":[2]}` |
| `pen.up()` | `"up"` | `[]` | `{"cmd":"up","args":[]}` |
| `pen.down()` | `"down"` | `[]` | `{"cmd":"down","args":[]}` |
| `pen.o(r)` | `"o"` | `[r]` | `{"cmd":"o","args":[50]}` |
| `pen.oo(r)` | `"oo"` | `[r]` | `{"cmd":"oo","args":[50]}` |
| `pen.r(w,h)` | `"r"` | `[w,h]` | `{"cmd":"r","args":[100,60]}` |
| `pen.rr(w,h)` | `"rr"` | `[w,h]` | `{"cmd":"rr","args":[100,60]}` |
| `pen.moveTo(x,y)` | `"moveTo"` | `[x,y]` | `{"cmd":"moveTo","args":[0,100]}` |
| `pen.lineTo(x,y)` | `"lineTo"` | `[x,y]` | `{"cmd":"lineTo","args":[200,100]}` |

### 注意

- 控制流（`for`、`if/else`）和变量操作**不产生轨迹记录**，只有 pen 方法调用产生记录
- 链式调用中每个方法**分别产生一条轨迹记录**
- `args` 中所有值为执行时的实际计算结果（变量已展开）

## 三、判题逻辑

### 比对规则

1. **长度一致**：用户轨迹与期望轨迹的记录条数必须相同
2. **cmd 一致**：每条记录的 `cmd` 必须与期望完全一致（字符串相等）
3. **args 在容差内**：每个参数值满足 `|用户值 - 期望值| ≤ 容差`

### 默认容差

| 参数类型 | 容差 |
|----------|------|
| 距离（fd/bk/moveTo/lineTo 的坐标值） | ±2 |
| 角度（rt/lt） | ±1° |
| 颜色（c） | 0（精确匹配） |
| 线宽（size） | 0（精确匹配） |
| 圆半径 / 矩形边长 | ±2 |

### 判题结果

- **Pass（通过）**：所有条目均满足上述规则
- **Fail（不通过）**：任意条目不满足，返回首个不匹配的位置和差异

## 四、执行器约定

### 坐标系

- 海龟起始位置：画布中心（0，0）
- 初始朝向：正北（向上）
- `rt(n)`：顺时针转 n 度
- `lt(n)`：逆时针转 n 度

### 坐标转换（画布渲染）

GOC 坐标 → Canvas 坐标：
- `canvas_x = center_x + goc_x`
- `canvas_y = center_y - goc_y`（y 轴方向相反）

前进方向计算（heading = 0 表示正北）：
- `dx = distance × sin(heading_rad)`
- `dy = -distance × cos(heading_rad)`
