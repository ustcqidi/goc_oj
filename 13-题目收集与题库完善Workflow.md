# 题目收集与题库完善 Workflow

本文档描述**题目收集 + 题库完善**的端到端流程：从外部题目来源到 sample 入库、模式提炼、出题、题库与 OJ 更新。与《11-产品持续优化》分离：11 专注产品优化思路讨论；本文档专注这一条工作流的具体步骤与产出。

---

## 一、流程总览

```
自动收集 sample → 理解 sample → 提炼模式更新文档 → 出题 → 更新题库
```

| 步骤 | 输入 | 动作 | 产出 | 依赖/文档 |
|------|------|------|------|-----------|
| **1. 自动收集 sample** | 题目 URL | Skill 抓取页面，抽取标题、题干、代码、图 | `problems/samples/编号-题目名/`（description.md + 可选 output.png） | 见 11 功能扩展「自动收集 sample」Skill；题目链接来源可维护在 **`problems/题目来源清单-noi.hnai.net.md`**（含状态：待抓取/已抓取） |
| **2. 理解 sample** | 新 sample 的 description.md（+ output.png） | 分析图形、核心指令、考察点、数学对应；可选：补写/校验 solution | 单题分析（可结构化为 JSON 或段落），为步骤 3 提供输入 | 06 中「逐题分析」的写法；01 指令规范 |
| **3. 提炼模式更新文档** | 当前 samples 集合 + 步骤 2 的分析 | 归纳题干结构、题型分类、常考指令组合、与数学对应；追加或更新「出题模式小结」 | 更新 **06-GOC 题目案例与模式提炼.md** | 06 现有结构；08 标签/难度可反哺 |
| **4. 出题** | 06 模式 + 07 Schema + 08 分类 + 09 出题指南 + 01 指令 | 按规范生成新题（题干多样、不照抄 06），含 title、description、difficulty、tags、starterCode、solution | 新增题目 JSON 文件（如 `problems/016-xxx.json`） | 07/08/09；可批量或单题 |
| **5. 更新题库** | 新增或修改后的 `problems/*.json` | 更新 **problems/index.json**；若 OJ 用内嵌数据则生成 **oj/data/problems.js**（或通过脚本从 problems/ 生成）；可选：为题目补 expectedTrajectory（运行 solution 写回 JSON） | 题库与 OJ 数据一致，新题在列表中可见、可做、可判题 | 07 Schema；验收报告「题目与 OJ 同步」 |

---

## 二、说明

- **步骤 1～3**：把外部题目变成可复用的知识（沉淀到 06）。
- **步骤 4～5**：用知识生产题目并进入系统（题库 + OJ）。
- **实现方式**：可做成 Skill/Agent 链（收集 Skill → 理解+提炼 → 出题提示词/Agent → 更新题库脚本）；也可人工执行其中某几步。
- **自动化程度**：若「理解 sample」与「提炼模式」自动化，每新增一个 sample 后可选「是否立即更新 06」；出题可定时或按需（如「再生成 5 道进阶题」）。
- **一次性交给 Claude Code 完成五步**：见 **《16-ClaudeCode-题目收集与题库完善-全量任务说明.md》**，内含完整步骤、格式、编号与验收，复制文档中的「一句话任务」即可零干预运行。
