## 三、GOC-006：画布与目标图形居中、防截断

### 3.1 目标

- 主画布（运行结果）与「目标图形」canvas 内的图形**居中显示**，且**不截断**（轨迹超出时整体缩放以适应画布）。
- 两处使用**同一套**「根据轨迹适应画布」的逻辑，保证表现一致。

### 3.2 实现要点

1. **Bounding box**：根据轨迹数组计算包围盒（遍历轨迹中的 fd/bk/rt/lt/moveTo/lineTo/o/oo/r 等产生的坐标或范围）；若执行器/渲染层已有 bbox 则复用，否则在渲染前算一次并缓存。
2. **适应画布**：给定 canvas 宽高与留白（如 16px），计算  
   `scale = min((canvasW - 2*pad) / bboxW, (canvasH - 2*pad) / bboxH)`，以及使 bbox 中心对齐 canvas 中心的偏移 `(tx, ty)`。  
   若 bbox 宽高为 0（空轨迹或单点），则 scale=1 或仅做居中。
3. **渲染**：在绘制轨迹前 `ctx.save()`，`ctx.setTransform(scale, 0, 0, scale, tx, ty)`（或先 translate 再 scale），绘制后 `ctx.restore()`。**主画布**与**目标图形 canvas** 均调用同一套「计算 scale/tx/ty + 应用 transform 后渲染」的逻辑。
4. **坐标系**：与 10 规范一致（海龟原点 (0,0)、朝向等），仅增加一层「适应画布」的变换，不改变轨迹数据本身。

### 3.3 验收

- 选一道轨迹范围较大或较小的题（如方格网、螺旋），运行后主画布图形完整居中、无裁切。
- 目标图形区预览的图形同样完整居中、无裁切，且与主画布视觉比例一致（都适应各自 canvas）。

## 四、GOC-030：排版与留白、指令区默认收起

### 4.1 目标

- 题目描述 / 目标图形 / 编辑区 / 画布 之间**加大间距**，减轻拥挤感。
- **指令参考区**默认不展示大块按钮列表，仅保留「📖 指令」入口；点击后展开完整面板（保留分组：画笔/坐标/图形/控制），可再次点击收起。

### 4.2 实现要点

1. **区块间距**：上述主区块之间设置 margin 或 padding，建议 16px～24px（或通过 CSS 变量统一，便于 029 复用）。
2. **指令区**：默认状态为**收起**（不渲染或隐藏完整指令按钮列表）；页面上保留明显的「📖 指令」按钮/入口，点击后展开指令面板，再次点击或点击「收起」可关闭。展开后样式与分组可与现有一致，仅默认状态改为收起。
3. **可发现性**：确保「📖 指令」在工具栏或固定位置可见，文案清晰（如「指令」或「📖 指令」）。

### 4.3 验收

- 打开 OJ，首屏不显示大块指令按钮，仅见「📖 指令」入口；点击后展开指令面板，可插入代码并可收起。
- 题目描述、目标图形、编辑区、画布之间留白明显，不再显得拥挤。

## 五、GOC-029：整体视觉与专业感

### 5.1 目标

- 统一主色/辅色、字体层级、按钮与区块样式，提升「专业感」。
- 通过 **CSS 变量**（Design Token）集中管理，便于后续调整。

### 5.2 实现要点

1. **CSS 变量**：在 `oj/index.html` 的 `<style>` 内或新建 `oj/css/tokens.css` 中定义，例如：  
   `--goc-primary`、`--goc-secondary`、`--goc-bg`、`--goc-border`、`--goc-text`、`--goc-text-muted`；  
   `--goc-font-title`、`--goc-font-body`、`--goc-font-small`；  
   `--goc-radius`、`--goc-space-md` 等。数值可先与当前视觉效果接近，再微调为更统一的风格。
2. **按钮**：主操作（运行、提交）使用主色实心；次操作（重置、提示等）使用 outline 或浅底；统一圆角（如 `var(--goc-radius)`）。
3. **区块**：左侧题目列表、中间题干、右侧编辑区使用统一的背景与 padding（如 `var(--goc-space-md)`），必要时 1px 分隔线。
4. **范围**：仅改 OJ 单页（goc/oj/），不扩大至其他目录；若当前样式为内联，可逐步替换为使用变量的 class，避免一次大改导致布局错乱。

### 5.3 验收

- 全页主色、字号、圆角、留白统一，无明显「拼凑感」。
- 运行 / 提交 / 重置 / 提示 等按钮样式区分清晰且一致。
