## 实现要点

1. **布局**  
   - 题目描述（prob-desc / p-desc）与目标图形（sol-preview、sol-canvas-wrap、sol-canvas）同属题目面板（prob-panel），当前多为固定高度或比例。  
   - 改为：两区块共占一列，**比例可调**。可选方案：  
     - **方案 A**：使用 CSS `flex` 或 `grid`，为「描述区」与「目标图形区」设置 `flex: 1` 或 `fr` 比例（如 1:1 或 4:6），通过一个 CSS 变量或 class 切换 50/50、40/60 等预设比例。  
     - **方案 B**：在两者之间增加**可拖拽分隔条**（resizer），拖动时改变描述区与目标图形区的宽度或高度占比，更新 canvas 的 width/height 后触发重绘。

2. **Canvas 与 fitMode**  
   - 目标图形区尺寸变化时，`sol-canvas` 的宽高需随之更新（或容器尺寸变化时读取 clientWidth/clientHeight 赋给 canvas）。  
   - 现有 GOC-006 的「适应画布」逻辑（computeBBox、computeFitTransform、fitMode）在渲染时使用 canvas 的 width/height；**尺寸变化后必须重新计算 transform 并重绘**，否则会拉伸模糊或留白不当。  
   - 在「选中题目后渲染目标图形」的逻辑中，若存在 resize 或拖拽事件，在尺寸变化后调用同一套渲染入口（传入 trajectory 或 solution 运行结果 + 新尺寸），保证居中、防截断仍然有效。

3. **体验**  
   - 默认比例建议使目标图形区不小于题目描述区（例如 50/50 或 40/60），避免一开始仍过小。  
   - 若有拖拽条，需在样式上易识别（如窄条、hover 变色），且不破坏移动端基本可用性（可仅做比例预设，不做拖拽）。
