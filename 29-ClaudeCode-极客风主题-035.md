## 实现要点

1. **色彩**  
   - 背景：主区域深色（如 #1e1e1e、#0d1117 等），侧栏可略深或同色；画布区背景保持深灰便于看清轨迹。  
   - 强调色：主操作按钮、链接、选中态用高对比色（如 #00ff88 绿、#00d4ff 青、#ff9500 橙），与深底搭配。  
   - 文字：正文浅灰/白（#e6edf3、#c9d1d9），次要文字更灰；错误/警告色在深色底上仍可辨。

2. **字体**  
   - 代码区、行号、控制区使用等宽字体（如 Consolas、Monaco、JetBrains Mono、'Fira Code'）；标题可保留非等宽或同用等宽统一 geek 感。

3. **与现有 token 衔接**  
   - 当前已有 --goc-primary、--goc-accent 等（GOC-029）；改为极客风时**重定义**这些变量为深色主题下的色值，或新增一组变量（如 --goc-geek-bg、--goc-geek-fg、--goc-geek-accent），在 body 或根容器上应用。  
   - 按钮、边框、输入框、侧栏 tab 等统一使用新变量，避免硬编码色值。

4. **可选：主题切换**  
   - 若保留「浅色/极客」双主题：通过 data-theme="geek" 或 class 切换根节点样式，默认可设为 geek。  
   - 若只做极客风：直接替换现有浅色变量为深色方案即可。

5. **画布与高亮**  
   - 主画布、目标图形 canvas 背景与轨迹颜色在深色背景下需清晰可见（轨迹色可用浅色或高对比色）。  
   - 代码高亮层（GOC-014）的配色改为深色背景下的方案（关键字、字符串、注释等仍可区分）。
