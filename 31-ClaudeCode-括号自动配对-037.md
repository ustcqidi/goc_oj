## 实现要点

1. **配对表**  
   - 左 → 右：`{`→`}`、`(`→`)`、`[`→`]`、`"`→`"`、`'`→`'`（可选）。  
   - 若输入为右半且光标前为对应左半，可视为「跳过右半」：按右半键时仅移动光标，不插入（可选，避免重复输入时出现双右半）。

2. **触发时机**  
   - 在 **keydown** 或 **keypress** 中拦截上述左半键：阻止默认插入，在光标位置插入「左半+右半」，然后将光标置于左半与右半之间（selectionStart = selectionEnd = 原位置 + 1）。  
   - 插入后触发 input 或手动调用高亮更新（highlightCode），保证高亮层与内容一致。

3. **「跳过插入」规则**  
   - 若当前光标右侧**紧邻**字符已是本应自动补的右半（且未选中文字），则视为用户按了「跳过」：只将光标右移一位，不插入新字符。  
   - 例如：输入 `{` 后已是 `{|}`，用户再按 `}` 时只移动光标到 `{}|`，不变成 `}}`。

4. **选区处理**  
   - 若有选中文字：输入左半时可在选区两侧插入左右半，即「选中的内容被括号包住」，光标落在右半前（可选，常见 IDE 行为）。  
   - 若无选中：按上述「左半+右半、光标居中」处理。

5. **与现有逻辑兼容**  
   - 不破坏 GOC-014 语法高亮、GOC-015 补全、GOC-016 格式化；插入字符后需触发高亮更新，格式化仍基于完整内容。  
   - 若补全或粘贴会插入多字符，配对逻辑只针对**单键输入**的上述符号即可。

6. **引号特殊**  
   - `"` 与 `'`：成对插入时与 `{}` 类似；若字符串内再输入引号，可只插入一个（由后续逻辑或用户自行处理），避免过度智能导致字符串被截断。简单实现：每次输入 `"` 都插入 `""` 且光标居中；若遇到「光标后已是 `"`」则只移动光标。
