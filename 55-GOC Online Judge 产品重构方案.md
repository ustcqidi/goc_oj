# GOC Online Judge 产品重构方案

> 可直接交付开发

---

# 一、产品目标

## 1.1 核心定位
将当前系统从：
- 传统 OJ 工具

升级为：

👉 **可视化编程学习平台（图形 + 反馈驱动）**

---

## 1.2 核心体验目标

1. 写代码更专注
2. 看结果更清晰
3. 学习路径更自然
4. 图形结果可观察、可缩放、可拖动

---

# 二、当前问题总结

## 2.1 布局问题
- 左侧题库长期占用空间
- 编辑器空间不足
- 功能区混杂
- 输出区域与编辑器争抢空间，主次不清

## 2.2 交互问题
- 画布不可缩放
- 输出区域过小
- 画布尺寸固定，窗口变化后不能自适应
- 缺少反馈（运行无状态）

## 2.3 信息层级问题
- 所有按钮权重相同
- 高频操作不突出
- 图形输出、文本输出、错误输出未分层

---

# 三、整体结构设计

## 3.1 页面结构（三层模型）

```
顶部：题目信息
中部：核心工作区
底部：运行反馈
```

---

## 3.2 中部工作区布局

桌面端采用双列布局：

```
------------------------------------------------------------
#072 多边形环    [标签] [难度] [进度]
------------------------------------------------------------

| 题目 / 指令区 | 代码编辑器                 | 图形输出区 |
|               |                           |           |
|               |                           |           |

------------------------------------------------------------
[运行] [提交] [格式化]                  [适应窗口] [100%]
------------------------------------------------------------
[图形] [输出] [错误]
------------------------------------------------------------
```

移动端或窄屏降级为上下布局：

```
题目
编辑器
输出
```

---

## 3.3 空间分配原则

- 编辑器为主工作区，默认占中部宽度 `55% ~ 60%`
- 图形输出区默认占中部宽度 `40% ~ 45%`
- 题目 / 指令区默认收起或窄栏显示
- 输出区不得低于最小可用尺寸，否则自动切换为上下布局

### 默认尺寸建议

- 桌面端：图形区最小宽度 `420px`
- 桌面端：图形区最小高度 `320px`
- 窄屏：图形区默认高度 `38vh`，最小高度 `280px`
- 用户拖拽后的布局比例需要持久化

# 4.3 输出（画布系统）⭐核心

## 4.3.1 设计目标

输出区需要从“附属结果区域”升级为“核心观察区域”。

要求：

- 默认看得清
- 放大后不失真
- 拖动后不迷路
- 窗口变化后能自适应
- 图形输出、文本输出、错误输出彼此独立

---

## 4.3.2 可调整布局

支持拖拽分割：

```
| 编辑器 | 图形输出 |
```

### 交互要求

- 用户可拖拽分隔条实时调整编辑器与输出区比例
- 停止拖拽后保存比例到本地
- 当图形区宽度小于阈值时，自动切换为上下布局
- 切换题目时不重置用户最近一次布局偏好

---

## 4.3.3 全屏模式

按钮：

```
[🔍 全屏]
```

### 规则

- 全屏模式仅聚焦图形输出
- `ESC` 退出
- 退出后恢复进入前的缩放比例与视图位置
- 全屏状态下仍可缩放、拖动、重置

---

## 4.3.4 缩放能力

支持：

- `Ctrl + 滚轮`
- 触控板双指缩放
- `+ / -` 按钮
- 预设倍率按钮
- `适应窗口`

### 视图控制

```
[适应窗口] [100%] [150%] [200%] [重置]
```

### 交互细节

- 缩放中心优先以鼠标所在位置为中心
- 首次运行默认执行一次 `适应窗口`
- 支持的缩放范围建议为 `50% ~ 400%`
- 当前倍率需要可见，例如：`125%`
- 当图形已被拖离可视区时，`重置` 可恢复到初始可视状态

### 技术要求

必须使用：

```js
ctx.scale(...)
```

或等价的 canvas 变换能力：

```js
ctx.setTransform(...)
```

禁止使用 DOM/CSS 缩放画布内容，例如：

```css
transform: scale(...)
```

### 原因

- DOM 缩放会导致坐标系与绘图逻辑脱节
- 鼠标命中、拖动、坐标提示会失真
- 线条与文字在高倍率下容易模糊

---

## 4.3.5 拖动（Pan）

- 鼠标按下并拖动画布可平移视图
- 拖动仅改变视图，不修改原始绘图数据
- 拖动后保留当前位置，直到用户点击 `重置` 或 `适应窗口`

### 额外要求

- 拖动时显示抓手态光标
- 缩放倍率为 `100%` 时仍允许轻量平移
- 平移与缩放状态需要统一管理

---

## 4.3.6 动态尺寸与清晰度

这是本次重构的 P0。

### 必须满足

- 输出区尺寸跟随容器变化实时重绘
- 浏览器窗口变化后，不允许出现画布内容被裁切但不重绘
- 切换侧边栏、全屏、布局比例后，画布立即重新计算可视区域
- 高分屏下保持清晰，不能发虚

### 技术建议

- 使用 `ResizeObserver` 监听画布容器尺寸
- 结合 `devicePixelRatio` 设置实际 canvas 像素尺寸
- 视图状态统一维护：

```js
{
  scale,
  offsetX,
  offsetY
}
```

- 每次尺寸变化后按如下顺序重绘：

```js
reset transform
apply dpr
apply pan / zoom
redraw scene
```

# 4.4 输出结构

改为 tab：

```
[图形] [输出] [错误]
```

### 默认行为

- 默认优先显示 `图形`
- 有报错时自动切换到 `错误`
- 纯文本题目可默认切到 `输出`

### 说明

- `图形`：Canvas 可视结果
- `输出`：标准输出、调试信息
- `错误`：语法错误、运行时错误、定位信息

```

```

### 规则

# 五、交互流程设计

## 5.1 主流程

1. 阅读题目
2. 编写代码
3. 点击运行
4. 查看图形或文本结果
5. 需要时放大、拖动、重置视图

---

## 5.2 自动行为

- 点击运行 → 自动滚动到输出区域
- 首次运行 → 默认切到 `图形` tab
- 错误 → 自动切到 `错误` tab 并高亮代码行
- 成功运行且图形超出可视区域 → 自动执行 `适应窗口`

# 十、技术建议

## 10.1 前端

- React / Vue
- SplitPane（布局）
- Canvas（绘制）
- ResizeObserver（尺寸监听）

---

## 10.2 状态管理

- 当前题目
- 编辑器内容
- 输出状态
- 当前输出 tab
- 视图状态（缩放 / 平移 / 全屏）

---

## 10.3 画布实现建议

- 维护独立的 scene 数据与 view 数据
- scene 负责真实绘图对象
- view 负责 `scale / offset / viewport`
- 所有重绘通过统一 render 函数执行
- 不允许把缩放逻辑散落到业务指令中

---

# 十二、验收标准

## 12.1 画布缩放验收

- 用户可通过滚轮和按钮完成缩放
- 缩放后画面不模糊、不跳变
- 缩放中心符合预期
- 重置后可回到初始视图

## 12.2 动态布局验收

- 拖拽分栏后布局即时生效
- 刷新页面后保留最近一次比例
- 窗口缩小时不会把输出区压到不可用

## 12.3 输出体验验收

- 首次运行后用户能直接看到图形结果
- 报错时能快速看到错误信息
- 图形、文本、错误三类信息互不干扰

用户体验：

👉 写代码更专注  
👉 看结果更清晰
