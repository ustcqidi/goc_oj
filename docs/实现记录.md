## 各功能验证方法

### A1：sync-problems.js

```bash
# 在 goc/ 目录下运行
node scripts/sync-problems.js
```

预期输出：打印题目数量统计，`problems/index.json` 和 `oj/data/problems.js` 被更新。

### A2：动画绘制 + 速度控制

1. 打开 `oj/index.html`，选择任意题目
2. 在工具栏看到速度选择器：慢 / 正常 / 快 / 极速
3. 选"慢"后点击"运行"：画布应逐步绘制，能看到动画效果
4. 选"极速"后点击"运行"：立即完整渲染（无动画）
5. 动画进行中再次点击"运行"：上一次动画被取消，重新开始

---

### A3：错误行号追踪 + 高亮

1. 在代码编辑器中写入有语法错误的代码，例如：
   ```
   pen.c(1).size(2)
   pen.fd(100).rt(90);
   ```
   （第1行缺少分号）
2. 点击"运行"
3. 结果栏显示 `❌ 第N行：缺少分号 `;`` 格式的中文错误
4. 编辑器中第N行被选中高亮，并滚动到该行

---

### A4：判题失败原因显示

1. 选一道题，故意写错代码（例如少画一步）
2. 点击"提交"
3. 结果栏显示红色 `❌ 未通过：` 后跟具体原因，如"轨迹长度不一致：期望 8 步，你的代码执行了 4 步"

---

### A5：Tag 标签筛选

1. 侧边栏难度筛选按钮下方出现标签 chip 行
2. 点击某个 tag（如"循环"）：题目列表过滤为仅含该 tag 的题目
3. 同时可与难度筛选叠加（AND 逻辑）
4. 点击"全部标签"恢复全量显示

---

### A6：指令参考面板

1. 编辑器工具栏有"📖 指令"按钮
2. 点击后在工具栏与编辑器之间展开参考面板
3. 面板顶部显示 0-15 号颜色色块
4. 下方按类别列出所有 pen 指令，每个有名称、描述和"插入"按钮
5. 点击"插入"：代码片段被插入到当前光标位置
6. 再次点击"📖 指令"：面板收起

---

### A7：目标图形预览

1. 选择题目后，题目描述下方出现"目标图形"区域
2. canvas 自动渲染题目的 solution 轨迹（参考效果图）
3. 点击"收起"按钮：canvas 隐藏，按钮变为"展开"
4. 点击"展开"：canvas 重新显示

## 技术说明

### goc-executor.js 新增 API

```javascript
// 绘制背景网格和十字轴（不清除画布外的状态）
GOCRenderer.drawBackground(ctx, w, h)

// 创建初始 turtle 状态对象
GOCRenderer.createState()
// 返回: { x:0, y:0, heading:0, penDown:true, color:'#222222', lw:1 }

// 执行单步轨迹，更新 state 并绘制
GOCRenderer.applyStep(ctx, step, state, w, h)

// 完整渲染（= drawBackground + createState + 循环 applyStep）
GOCRenderer.render(ctx, trajectory, w, h)

// 颜色数组（16色）
GOCRenderer.GOC_COLORS
```

### 错误对象格式

```javascript
// GOCExecutor.run() 返回
{ ok: false, error: '第 3 行：缺少分号 `;`', line: 3, trajectory: [] }
{ ok: true,  error: null, line: null, trajectory: [...] }
```
